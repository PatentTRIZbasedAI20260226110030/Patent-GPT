## 1. 사용자 플로우

```
[랜딩/홈] → [문제 입력] → [결과 로딩] → [결과 화면] → [DOCX 다운로드]
                  │
                  └── (선택) [선행특허 검색]
```

| 단계 | 화면 | 설명 |
| :--: | :-- | :-- |
| 1 | 랜딩 | 서비스 소개, CTA로 문제 입력 화면 유도 |
| 2 | 문제 입력 | 키워드/문제 기술, 기술 분야(선택), 생성 실행 |
| 3 | 결과 로딩 | 진행 상태 표시 (TRIZ 분류 → 검색 → 추론 → 초안 생성) |
| 4 | 결과 화면 | 특허 아이디어, TRIZ 원리, 선행특허, 추론 흐름 표시 |
| 5 | DOCX 다운로드 | 생성된 특허 초안 파일 다운로드 |
| (부가) | 선행특허 검색 | 검색만 단독 실행 (선택 기능) |

## 2. 화면 목록

| 화면 ID | 화면명 | 경로(가정) | 우선순위 |
| :--: | :-- | :-- | :--: |
| S-01 | 랜딩 | `/` | 높음 |
| S-02 | 문제 입력 | `/generate` | 높음 |
| S-03 | 결과 로딩 | `/generate` (같은 페이지, 상태 전환) | 높음 |
| S-04 | 결과 화면 | `/result/:id` 또는 `/generate` 내 결과 영역 | 높음 |
| S-05 | 선행특허 검색 | `/search` | 중간 |
| S-06 | 에러/빈 결과 | 공통 컴포넌트 | 중간 |

## 3. 화면별 상세 정의

### S-01 랜딩

**목적:** 서비스 소개, 사용자 진입 유도

| 항목 | 내용 |
| :-- | :-- |
| **헤더** | 로고, 서비스명 "Patent-GPT", (선택) 로그인/회원가입 |
| **히어로** | "TRIZ 기반 특허 아이디어 발굴" 메시지, 핵심 가치 1줄 |
| **CTA** | "특허 아이디어 생성하기" → `/generate` 이동 |
| **플로우 요약** | `키워드 입력 → 아이디어 생성 → 가치 평가` 3단계 아이콘/텍스트 |
| **푸터** | 팀 정보, 라이선스, (선택) API 문서 링크 |

**Figma 디자인 포인트:**
- 심플한 1-column 레이아웃
- CTA 버튼 강조 (브랜드 컬러)
- 데스크톱/모바일 반응형 고려

---

### S-02 문제 입력

**목적:** 사용자 문제·키워드를 입력하고 생성 요청

| 영역 | UI 요소 | 필수 | 설명 |
| :-- | :-- | :--: | :-- |
| 상단 | 제목 | ✓ | "특허 아이디어 생성" |
| 입력 | 문제 설명 (textarea) | ✓ | `problem_description`, placeholder 예: "발열은 줄이고 싶지만 두께는 얇아야 한다" |
| 입력 | 기술 분야 (select/input) | | `technical_field`, 예: 전자기기, 의료기기 |
| 입력 | 최대 회피 시도 횟수 (number) | | `max_evasion_attempts`, 기본 3, 1~10 |
| 액션 | "생성하기" 버튼 | ✓ | 클릭 시 S-03 로딩 상태로 전환 |

**유효성 검사:**
- 문제 설명: 1자 이상 필수
- 버튼 비활성화 조건: 문제 설명 비어 있음

**Figma 디자인 포인트:**
- 폼 레이아웃, 라벨·입력 필드 명확히 구분
- textarea 최소 3~4줄 높이
- 로딩 중 버튼 비활성화/스피너 표시 상태 별도 목업

---

### S-03 결과 로딩

**목적:** 4단계 파이프라인 진행 상황 표시

| 단계 | 표시 텍스트 | 설명 |
| :--: | :-- | :-- |
| 1 | TRIZ 원리 분류 중... | 문제를 TRIZ 원리에 매핑 |
| 2 | 선행 특허 검색 중... | BM25 + 벡터 검색 + 리랭킹 |
| 3 | 추론 및 회피 설계 검토... | 유사도 높을 때 재설계 |
| 4 | 특허 초안 생성 중... | DOCX 생성 |

**UI 요소:**
- 단계별 체크리스트 또는 프로그레스 스텝
- (선택) `reasoning_trace` 실시간 스트리밍 표시
- 취소 버튼 (가능하다면)

**Figma 디자인 포인트:**
- 4단계 스텝 UI (수직 또는 수평)
- 로딩 스피너/스켈레톤
- 예상 소요 시간 안내 문구 (예: "보통 30초~1분 소요")

---

### S-04 결과 화면

**목적:** 생성된 특허 아이디어, TRIZ 원리, 선행특허, 추론 흐름 표시

| 영역 | 데이터 | UI 타입 |
| :-- | :-- | :-- |
| **특허 초안** | `patent_draft` | 아코디언 또는 탭 |
| ↳ 발명의 명칭 | `title` | 제목(H1) |
| ↳ 요약 | `abstract` | 단락 텍스트 |
| ↳ 배경기술 | `background` | 단락 텍스트 |
| ↳ 해결과제 | `problem_statement` | 단락 텍스트 |
| ↳ 해결수단 | `solution` | 단락 텍스트 |
| ↳ 청구항 | `claims` (목록) | 번호 리스트 |
| ↳ 발명의 효과 | `effects` | 단락 텍스트 |
| **TRIZ 원리** | `triz_principles` | 카드 그리드 또는 리스트 |
| ↳ 각 원리 | `number`, `name_ko`, `name_en`, `description` | 카드 1장 |
| **선행 특허** | `similar_patents` | 테이블 또는 카드 |
| ↳ 각 특허 | `title`, `abstract`, `application_number`, `similarity_score` | 행 1개 |
| **추론 흐름** | `reasoning_trace` | 타임라인 또는 리스트 |
| **다운로드** | `docx_download_url` | "DOCX 다운로드" 버튼 |

**액션:**
- "다시 생성" → S-02로 이동
- "DOCX 다운로드" → `GET /api/v1/patent/{draft_id}/docx` 호출 후 파일 다운로드

**Figma 디자인 포인트:**
- 스크롤 가능한 단일 페이지 또는 탭 구분
- TRIZ 원리 카드: 번호, 한글명, 설명 일부
- 선행특허: 유사도 % 표시 (예: 85%)
- 모바일에서 카드 스택 레이아웃

---

### S-05 선행특허 검색

**목적:** 아이디어 생성 없이 선행 특허 검색만 수행

| 영역 | UI 요소 | 설명 |
| :-- | :-- | :-- |
| 입력 | 검색어 (input) | `query`, placeholder: "방열 구조" |
| 입력 | 결과 개수 (number) | `top_k`, 기본 5, 1~50 |
| 액션 | "검색" 버튼 | `POST /api/v1/patent/search` 호출 |
| 결과 | `results` (SimilarPatent[]) | S-04 선행특허 영역과 동일 UI |

**Figma 디자인 포인트:**
- 심플한 검색 폼
- 결과가 없을 때 빈 상태 UI

---

### S-06 에러/빈 결과

**목적:** API 오류, 유효성 오류, 빈 결과 처리

| 케이스 | 메시지 예시 | UI |
| :-- | :-- | :-- |
| API 오류 | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." | 배너 + 재시도 버튼 |
| 유효성 오류 (422) | "문제 설명을 입력해 주세요." | 인라인 에러 메시지 |
| 빈 검색 결과 | "검색 결과가 없습니다." | 빈 상태 일러스트 + 안내 문구 |

**Figma 디자인 포인트:**
- 에러 배너 색상 (예: red/warning)
- 빈 상태 일러스트 또는 아이콘

## 4. 공통 UI 요소

| 요소 | 용도 |
| :-- | :-- |
| **헤더** | 로고, 내비게이션(홈, 생성, 검색), (선택) 로그인 |
| **푸터** | 팀 정보, 링크 |
| **버튼** | Primary(CTA), Secondary(보조), Ghost(텍스트) |
| **입력 필드** | text, textarea, number, select |
| **로딩** | 스피너, 스켈레톤, 프로그레스 스텝 |
| **카드** | TRIZ 원리, 선행특허 등 콘텐츠 블록 |

## 5. 데이터 스키마

### 생성 요청 (POST /api/v1/patent/generate)

```json
{
  "problem_description": "string (필수)",
  "technical_field": "string (선택)",
  "max_evasion_attempts": 3
}
```

### 생성 응답

```json
{
  "patent_draft": {
    "title": "string",
    "abstract": "string",
    "background": "string",
    "problem_statement": "string",
    "solution": "string",
    "claims": ["string", "..."],
    "effects": "string"
  },
  "triz_principles": [
    {
      "number": 1,
      "name_en": "string",
      "name_ko": "string",
      "description": "string"
    }
  ],
  "similar_patents": [
    {
      "title": "string",
      "abstract": "string",
      "application_number": "string",
      "similarity_score": 0.85
    }
  ],
  "reasoning_trace": ["string", "..."],
  "docx_download_url": "data/drafts/patent_draft_xxx.docx"
}
```

### 검색 요청 (POST /api/v1/patent/search)

```json
{
  "query": "string",
  "top_k": 5
}
```

### 검색 응답

```json
{
  "results": [
    {
      "title": "string",
      "abstract": "string",
      "application_number": "string",
      "similarity_score": 0.85
    }
  ]
}
```
